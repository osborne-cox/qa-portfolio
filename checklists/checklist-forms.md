# Чек-лист: Practice Form

**Страница:** https://demoqa.com/automation-practice-form
**Что это:** большая форма регистрации студента — текстовые поля, радиокнопки, чекбоксы, выпадающие списки, дата, загрузка файла, автодополнение и модалка с результатом.

Это не тест-кейсы с шагами, а список «что проверить» — то, что я прохожу глазами и руками перед тем, как заводить детальные кейсы. Отмечаю по ходу: `[ ]` — не проверено, `[x]` — проверено и ок.

Обязательные для отправки поля: First Name, Last Name, Gender, Mobile Number. Остальные — опциональны.

## Текстовые поля

- [ ] First Name — обязательное, пустое блокирует отправку (красная рамка)
- [ ] Last Name — обязательное, пустое блокирует отправку
- [ ] First/Last Name принимают буквы, кириллицу, дефис (Анна-Мария)
- [ ] First/Last Name — поведение при вводе цифр и спецсимволов (есть ли валидация формата)
- [ ] First/Last Name — поведение при очень длинном вводе (200+ символов), наличие maxlength
- [ ] Email — при заполнении валидируется формат (нет `@` → ошибка)
- [ ] Email — пустой email не блокирует отправку (поле необязательное)
- [ ] Current Address (textarea) — принимает многострочный текст, переносы строк
- [ ] Current Address — ведёт себя корректно при длинном тексте (растяжение/скролл, не ломает вёрстку)

## Радиокнопки Gender

- [ ] Gender обязателен — без выбора форма не отправляется
- [ ] Можно выбрать только один вариант (Male / Female / Other) — взаимоисключение
- [ ] Клик по подписи (label), а не только по кружку, тоже выбирает вариант

## Mobile Number

- [ ] Mobile обязателен — пустое поле блокирует отправку
- [ ] Принимает ровно 10 цифр; меньше 10 — невалидно
- [ ] Поведение при вводе более 10 цифр (обрезается или отклоняется)
- [ ] Не принимает буквы и спецсимволы (только цифры)

## Date of Birth

- [ ] По умолчанию подставлена текущая дата
- [ ] Открывается датапикер, выбор дня/месяца/года работает
- [ ] Смена месяца и года через выпадающие списки внутри пикера
- [ ] Выбор будущей даты (нужно зафиксировать ожидаемое поведение — допустима ли)
- [ ] Выбранная дата отображается в поле в нужном формате (dd MMM yyyy)

## Subjects (автодополнение)

- [ ] Ввод первых букв предмета показывает подсказки (Maths, Physics, ...)
- [ ] Можно добавить несколько предметов в виде тегов
- [ ] Тег предмета удаляется крестиком
- [ ] Ввод несуществующего предмета не добавляет тег

## Hobbies (чекбоксы)

- [ ] Можно выбрать несколько хобби одновременно (Sports, Reading, Music)
- [ ] Повторный клик снимает выбор
- [ ] Клик по подписи переключает чекбокс

## Picture (загрузка файла)

- [ ] Кнопка выбора файла открывает системный диалог
- [ ] Загрузка изображения (jpg/png) — имя файла отображается рядом
- [ ] Поведение при загрузке файла неизображения (txt, exe) — есть ли ограничение по типу
- [ ] Поведение при большом файле

## State & City (зависимые списки)

- [ ] State — выпадающий список открывается и позволяет выбрать регион
- [ ] City заблокирован/пуст, пока не выбран State
- [ ] Список City зависит от выбранного State (для NCR — Delhi, Gurgaon, Noida)
- [ ] После смены State ранее выбранный City сбрасывается

## Отправка и модалка результата

- [ ] Submit при всех заполненных обязательных полях открывает модалку «Thanks for submitting the form»
- [ ] В модалке данные совпадают с введёнными (все строки)
- [ ] Незаполненные опциональные поля в модалке отображаются пусто/корректно
- [ ] Submit с незаполненными обязательными — модалка не появляется, поля подсвечены красным
- [ ] Кнопка/клик вне модалки закрывает её

## Вёрстка и окружение

- [ ] Кнопка Submit доступна для клика и не перекрыта рекламными баннерами (см. BUG-001)
- [ ] Форма корректно отображается при разном масштабе/разрешении
- [ ] Tab-навигация проходит по полям в логичном порядке
- [ ] Обязательные поля визуально помечены (или это узнаётся только после Submit — отметить как замечание)
