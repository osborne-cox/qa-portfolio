# Чек-лист: Practice Form

**Страница:** https://demoqa.com/automation-practice-form
**Что это:** большая форма регистрации студента — текстовые поля, радиокнопки, чекбоксы, выпадающие списки, дата, загрузка файла, автодополнение и модалка с результатом.

Это не тест-кейсы с шагами, а список «что проверить» — то, что я прохожу глазами и руками перед тем, как заводить детальные кейсы. `[ ]` — не проверено, `[x]` — проверено и ок.

Обязательные для отправки поля: First Name, Last Name, Gender, Mobile Number. Остальные — опциональны.

## Текстовые поля

- [x] First Name — обязательное, пустое блокирует отправку (красная рамка)
- [x] Last Name — обязательное, пустое блокирует отправку
- [x] First/Last Name принимают буквы, кириллицу, дефис (Анна-Мария)
- [x] First/Last Name — поведение при вводе цифр и спецсимволов (есть ли валидация формата)
- [x] First/Last Name — поведение при очень длинном вводе (200+ символов), наличие maxlength
- [x] Email — при заполнении валидируется формат (нет `@` → ошибка)
- [x] Email — пустой email не блокирует отправку (поле необязательное)
- [x] Current Address (textarea) — принимает многострочный текст, переносы строк
- [x] Current Address — ведёт себя корректно при длинном тексте (растяжение/скролл, не ломает вёрстку)

## Радиокнопки Gender

- [x] Gender обязателен — без выбора форма не отправляется
- [x] Можно выбрать только один вариант (Male / Female / Other) — взаимоисключение
- [x] Клик по подписи (label), а не только по кружку, тоже выбирает вариант

## Mobile Number

- [x] Mobile обязателен — пустое поле блокирует отправку
- [x] Принимает ровно 10 цифр; меньше 10 — невалидно
- [x] Поведение при вводе более 10 цифр (обрезается или отклоняется)
- [x] Не принимает буквы и спецсимволы (только цифры)

## Date of Birth

- [x] По умолчанию подставлена текущая дата
- [x] Открывается датапикер, выбор дня/месяца/года работает
- [x] Смена месяца и года через выпадающие списки внутри пикера
- [x] Выбор будущей даты (нужно зафиксировать ожидаемое поведение — допустима ли)
- [x] Выбранная дата отображается в поле в нужном формате (dd MMM yyyy)

## Subjects (автодополнение)

- [x] Ввод первых букв предмета показывает подсказки (Maths, Physics, ...)
- [x] Можно добавить несколько предметов в виде тегов
- [x] Тег предмета удаляется крестиком
- [x] Ввод несуществующего предмета не добавляет тег

## Hobbies (чекбоксы)

- [x] Можно выбрать несколько хобби одновременно (Sports, Reading, Music)
- [x] Повторный клик снимает выбор
- [x] Клик по подписи переключает чекбокс

## Picture (загрузка файла)

- [x] Кнопка выбора файла открывает системный диалог
- [x] Загрузка изображения (jpg/png) — имя файла отображается рядом
- [x] Поведение при загрузке файла неизображения (txt, exe) — есть ли ограничение по типу
- [x] Поведение при большом файле

## State & City (зависимые списки)

- [x] State — выпадающий список открывается и позволяет выбрать регион
- [x] City заблокирован/пуст, пока не выбран State
- [x] Список City зависит от выбранного State (для NCR — Delhi, Gurgaon, Noida)
- [x] После смены State ранее выбранный City сбрасывается

## Отправка и модалка результата

- [x] Submit при всех заполненных обязательных полях открывает модалку «Thanks for submitting the form»
- [x] В модалке данные совпадают с введёнными (все строки)
- [x] Незаполненные опциональные поля в модалке отображаются пусто/корректно
- [x] Submit с незаполненными обязательными — модалка не появляется, поля подсвечены красным
- [x] Кнопка/клик вне модалки закрывает её

## Вёрстка и окружение

- [x] Кнопка Submit доступна для клика и не перекрыта рекламными баннерами (см. BUG-001)
- [x] Форма корректно отображается при разном масштабе/разрешении
- [x] Tab-навигация проходит по полям в логичном порядке
- [x] Обязательные поля визуально помечены (или это узнаётся только после Submit — отметить как замечание)
