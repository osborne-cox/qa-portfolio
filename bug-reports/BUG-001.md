# BUG-001 — Рекламные баннеры перекрывают кнопку Submit и нижние поля формы

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Компонент** | Practice Form (/automation-practice-form) |
| **Severity** | Major |
| **Priority** | Medium |
| **Тип** | UI / блокировка взаимодействия |
| **Окружение** | Firefox 151.0.2, Windows 11, 1920×1080, окно не на весь экран |
| **Воспроизводимость** | Часто (зависит от высоты окна и подгрузки рекламы) |

## Описание

На странице Practice Form встроены сторонние рекламные блоки (фиксированный баннер снизу и боковой блок), которые при определённой высоте окна накрывают нижнюю часть формы — в том числе кнопку **Submit** и выпадающие списки **State / City**. Клик по «видимой» кнопке Submit попадает по баннеру, а не по кнопке, и форма не отправляется. Для дропдаунов State/City часть опций оказывается под баннером.

Это классический случай «элемент видно, но кликнуть нельзя»: координаты кнопки перекрыты другим элементом с большим z-index.

## Шаги воспроизведения

1. Открыть https://demoqa.com/automation-practice-form в окне средней высоты (не развёрнутом на весь экран).
2. Дождаться полной загрузки страницы, включая рекламные блоки.
3. Заполнить обязательные поля (First Name, Last Name, Gender, Mobile Number).
4. Прокрутить к кнопке Submit и кликнуть по ней.

## Ожидаемый результат

Кнопка Submit полностью доступна для клика; рекламные блоки не перекрывают интерактивные элементы формы. Клик по Submit открывает модалку с результатом.

## Фактический результат

Нижний рекламный баннер перекрывает кнопку Submit (и/или поля State/City). Клик по области кнопки приходится на баннер, форма не отправляется, модалка не появляется. Визуально кнопка «есть», но взаимодействие блокируется.

## Обходной путь

- Прокрутить страницу так, чтобы Submit оказался выше баннера, либо развернуть окно на весь экран.
- В автотестах — `scrollIntoView` + клик через JS (`element.click()`), минуя перекрытие.

## Влияние

Для ручного пользователя — раздражение и ощущение «кнопка не работает». Для автоматизации — нестабильные (флакающие) тесты: обычный `click()` по координатам падает с `ElementClickInterceptedException`. Это, кстати, одна из причин, по которой demoqa так любят как тренажёр — учит обходить именно такие ситуации.

## Примечание

Сайт учебный и живёт за счёт этой самой рекламы, так что баг «вечный» и воспроизводится у многих. К отчёту приложить скриншот с подсвеченным перекрытием (DevTools → выделение элемента баннера поверх Submit).
