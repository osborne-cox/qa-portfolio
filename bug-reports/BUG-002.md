# BUG-002 — Text Box: нет ограничения длины и валидации полей Full Name / Address

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Компонент** | Text Box (/text-box) |
| **Severity** | Minor |
| **Priority** | Low |
| **Тип** | Валидация ввода / вёрстка вывода |
| **Окружение** | Firefox 151.0.2, Windows 11, 1920×1080 |
| **Воспроизводимость** | Всегда (100%) |

## Описание

Поля Full Name, Current Address и Permanent Address на форме Text Box не имеют ни ограничения длины (`maxlength`), ни какой-либо валидации содержимого. В них принимается:

- строка из 500+ символов;
- имя, состоящее только из цифр (`1234567890`);
- строка из одних пробелов (нет trim).

После Submit введённое значение без изменений выводится в блок Output. Длинная строка без переносов растягивает блок Output и ломает его вёрстку (горизонтальный скролл / выход за границы рамки). Единственное валидируемое поле на форме — Email; остальные принимают что угодно.

## Шаги воспроизведения

1. Открыть https://demoqa.com/text-box.
2. В поле Full Name вставить строку из 550 символов (например, 550 раз буква `A`).
3. В Email ввести валидный адрес `test@example.com`.
4. Нажать Submit.

## Ожидаемый результат

У текстовых полей есть разумный лимит длины (например, 50–100 символов для имени), ввод сверх лимита обрезается или отклоняется. Имя из одних цифр и строка из одних пробелов либо отклоняются, либо нормализуются. Блок Output не ломает вёрстку.

## Фактический результат

Все 550 символов принимаются и целиком выводятся в Output. Атрибут `maxlength` у полей отсутствует. Имя `1234567890` и строка из пробелов также принимаются без ошибок. Длинная строка растягивает блок Output за пределы рамки, появляется горизонтальный скролл.

## Дополнительные наблюдения

- Имя из одних цифр (TC-001-04) — принимается.
- Только пробелы (TC-001-06) — принимаются, нет trim; в Output появляется `Name :` с пустым на вид значением.
- HTML-подобный ввод (`<b>test</b>`) выводится как текст, а не интерпретируется — React экранирует вывод, так что это не XSS. Отмечаю отдельно, чтобы не выдавать за дыру в безопасности.

## Обходной путь

Со стороны пользователя — нет (форма просто принимает мусор). Исправление на стороне разработки: добавить `maxlength`, trim и базовую валидацию формата имени.

## Влияние

Низкое для учебного стенда, но на реальной форме это привело бы к мусорным данным в БД и потенциальным проблемам с вёрсткой/хранением сверхдлинных строк. Сгруппировано с кейсами TC-001-04, TC-001-05, TC-001-06.
