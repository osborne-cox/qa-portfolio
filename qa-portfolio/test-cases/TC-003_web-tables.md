# TC-003 — Web Tables (CRUD-таблица)

**Страница:** https://demoqa.com/webtables
**Объект тестирования:** таблица записей с кнопками Add / Edit / Delete и полем Search. Колонки: First Name, Last Name, Age, Email, Salary, Department, Action.
**Окружение:** Chrome 125 / Windows 10.

Предусловие для всех кейсов, кроме оговорённых: на странице по умолчанию три записи (Cierra Vega, Alden Cantrell, Kierra Gentry). Поле Search пустое, выбрана страница 1.

Форма добавления (модалка «Registration Form») содержит поля: First Name, Last Name, Email, Age, Salary, Department. Все шесть обязательны.

---

## TC-003-01 — Добавление записи с корректными данными

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный |
| **Предусловие** | Открыта /webtables |

**Тестовые данные:**
- First Name: `Danil`
- Last Name: `Petrov`
- Email: `danil.petrov@example.com`
- Age: `29`
- Salary: `120000`
- Department: `QA`

**Шаги:**
1. Нажать кнопку «Add».
2. Заполнить все поля модалки тестовыми данными.
3. Нажать Submit.

**Ожидаемый результат:** модалка закрывается, в таблице появляется новая строка с введёнными данными.

**Фактический результат:** соответствует ожидаемому. Появляется четвёртая запись `Danil / Petrov / 29 / ... / 120000 / QA`. Модалка закрывается.

**Статус:** Pass

## TC-003-03 — Удаление записи

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный |
| **Предусловие** | В таблице есть запись `Kierra Gentry` (строка 3 по умолчанию) |

**Шаги:**
1. В строке `Kierra Gentry` нажать иконку Delete (корзина, `id=delete-record-3`).
2. Проверить таблицу.

**Ожидаемый результат:** строка исчезает без подтверждения; остальные записи сохраняются.

**Фактический результат:** строка `Kierra Gentry` удаляется немедленно. Диалога подтверждения нет — удаление происходит в один клик.

**Статус:** Pass

> Наблюдение: удаление без подтверждения. Для учебного стенда нормально, для боевой системы — риск случайного удаления (стоило бы диалог «вы уверены?»). Фиксирую как замечание, не как баг.

## TC-003-06 — Редактирование существующей записи

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный |
| **Предусловие** | В таблице есть запись `Cierra Vega` |

**Тестовые данные:** новое значение Salary = `999999`.

**Шаги:**

**Фактический результат:** модалка открывается, поля заполнены текущими значениями записи. После правки Salary и Submit строка в таблице показывает `999999`. Остальные поля не изменились.

**Статус:** Pass

## TC-003-07 — Валидация поля Age (нечисловой ввод)

| | |
|---|---|
| **Приоритет** | Medium |
| **Тип** | Негативный / граничный |
| **Предусловие** | Открыта /webtables |

**Тестовые данные:** Age = `abc`, Salary = `-500`, остальные поля валидны.

**Шаги:**
1. Нажать «Add».
2. Заполнить First/Last/Email/Department валидно.
3. В Age ввести `abc`, в Salary — `-500`.
4. Нажать Submit.

**Ожидаемый результат:** Age принимает только целые положительные числа; буквы и отрицательная зарплата отклоняются.

**Фактический результат:** поля Age и Salary не проверяют тип/диапазон содержимого — отклоняется только пустое поле. Нечисловой возраст и отрицательная зарплата проходят, если поля непустые. Числовой валидации нет.

**Статус:** Fail — отсутствует типовая/диапазонная валидация числовых полей.
