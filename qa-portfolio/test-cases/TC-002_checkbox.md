# TC-002 — Компонент «Check Box»

**Страница:** https://demoqa.com/checkbox
**Объект тестирования:** дерево чекбоксов с раскрытием/сворачиванием узлов, тремя состояниями (выбран / не выбран / частично выбран) и блоком результата.
**Окружение:** Chrome 125 / Windows 10.

Структура дерева для справки:

```
Home
├── Desktop
│   ├── Notes
│   └── Commands
├── Documents
│   ├── WorkSpace (React, Angular, Veu)
│   └── Office   (Public, Private, Classified, General)
└── Downloads
    ├── Word File.doc
    └── Excel File.doc
```

Логика выбора каскадная: отметка родителя выбирает всех потомков; выбор части потомков переводит родителя в промежуточное (half-check) состояние.

---

## TC-002-01 — Раскрытие и сворачивание дерева

| | |
|---|---|
| **Приоритет** | Medium |
| **Тип** | Позитивный / UI |
| **Предусловие** | Открыта /checkbox, дерево свёрнуто (виден только Home) |

**Шаги:**
1. Нажать кнопку «Expand all» (иконка с двойной стрелкой над деревом).
2. Убедиться, что раскрылись все узлы до листьев.
3. Нажать «Collapse all».
4. Свернуть обратно через переключатель `+`/`−` у узла Home.

**Ожидаемый результат:** Expand all раскрывает всё дерево; Collapse all сворачивает до Home; одиночный тоггл `+`/`−` раскрывает/сворачивает конкретный узел.

**Фактический результат:** соответствует ожидаемому. Все три способа управления раскрытием работают.

**Статус:** Pass

## TC-002-02 — Выбор одного листа

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный |
| **Предусловие** | Дерево раскрыто, ничего не выбрано |

**Шаги:**
1. Поставить чекбокс на листе `Notes` (внутри Desktop).
2. Посмотреть блок результата под деревом.

**Ожидаемый результат:** выбран только `Notes`; в результате перечислен один элемент, соответствующий этому листу.

**Фактический результат:** чекбокс `Notes` отмечен. Появляется строка «You have selected :» со значением `notes`. Совпадает с ожиданием.

**Статус:** Pass

> Наблюдение: в блоке результата выводятся не человекочитаемые подписи, а внутренние ключи узлов в нижнем регистре (например, `Word File.doc` → `wordFile`). Это не ошибка функционала, но несоответствие подписи и вывода — кандидат в минорный UX-дефект.

## TC-002-03 — Выбор родителя выбирает всех потомков

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный |
| **Предусловие** | Дерево раскрыто, ничего не выбрано |

**Шаги:**
1. Отметить чекбокс узла `Desktop`.
2. Проверить состояние дочерних `Notes` и `Commands`.
3. Проверить блок результата.

**Ожидаемый результат:** при выборе `Desktop` автоматически отмечаются `Notes` и `Commands`; в результате перечислены `desktop`, `notes`, `commands`.

**Фактический результат:** каскад срабатывает — оба потомка отмечены, в результате присутствуют все три значения.

## TC-002-04 — Промежуточное (half-check) состояние родителя

| | |
|---|---|
| **Приоритет** | High |
| **Тип** | Позитивный / граничный |
| **Предусловие** | Дерево раскрыто, ничего не выбрано |

**Шаги:**
1. Раскрыть `Documents` → `WorkSpace`.
2. Отметить только `React` (не трогая `Angular` и `Veu`).
3. Посмотреть на иконку чекбокса `WorkSpace` и выше — `Documents`, `Home`.

**Ожидаемый результат:** `WorkSpace`, `Documents` и `Home` отображаются в промежуточном состоянии (квадрат/чёрточка), т.к. выбрана только часть потомков.

**Фактический результат:** `WorkSpace`, `Documents`, `Home` показывают индикатор частичного выбора (half-check), а не полную галочку. Каскад вверх по дереву отрабатывает корректно.

**Статус:** Pass

## TC-002-05 — Выбор корня (Home) отмечает всё дерево

| | |
|---|---|
| **Приоритет** | Medium |
| **Тип** | Позитивный |
| **Предусловие** | Дерево свёрнуто или раскрыто, ничего не выбрано |

**Шаги:**
1. Отметить чекбокс `Home`.
2. Раскрыть дерево и проверить, что все узлы отмечены.
3. Проверить блок результата.

**Ожидаемый результат:** выбраны все узлы дерева (16 элементов, включая ветви и листья); в результате перечислены все значения.

**Фактический результат:** все узлы получают галочку, в результате перечисляются все листья и ветви. Список в результате длинный, но полный.

**Статус:** Pass

## TC-002-06 — Снятие выбора и очистка результата

| | |
|---|---|
| **Приоритет** | Medium |
| **Тип** | Позитивный |
| **Предусловие** | Выбран хотя бы один элемент (например, после TC-002-02) |

**Шаги:**
1. Снять единственную поставленную галочку (`Notes`).
2. Посмотреть блок результата.

**Ожидаемый результат:** при отсутствии выбранных элементов блок «You have selected» исчезает (или становится пустым).

**Фактический результат:** после снятия последнего чекбокса блок результата пропадает. Состояние возвращается к исходному.

**Статус:** Pass
